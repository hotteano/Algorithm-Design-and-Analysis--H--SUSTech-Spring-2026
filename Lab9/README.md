# Lab 9B: String Folding / 字符串折叠

## 问题描述 (Problem Description)

Tom 需要使用更少的字符来表示一个按特定规律重复的长字符串。
- **B**: 标记重复部分的起点（存档点/锚点）。如果是字符串的开头，第一个 `B` 可以省略。
- **R**: 将从上一个最近的 `B`（或开头）到当前位置之前的所有字符**复制（翻转）一倍**。

**目标**：求完全折叠该字符串后的最小长度。

## 算法分析：区间 DP (Interval DP)

由于一段字符串被单独存放在两个 `B` 之间，或者一个 `B` 与末尾之间是独立生成的，非常符合**区间 DP**的特征。

问题的核心在于处理 `B` 带来的影响：`R` 只能复制从**最近的一个 `B`** 开始的全部字符。为了消除这种上下文依赖，我们在 DP 状态中区分“区间内部是否有新加的 `B`”。

### 1. 状态定义

设原字符串为 $S$，长度为 $N$。
我们定义三维 DP 数组 `dp[i][j][0/1]`：

- `dp[i][j][1]`：表示生成子串 $S[i...j]$，且**该区间内部不包含任何新的 `B`**（即只依赖于 $i$ 位置之前已经存在的 `B` 或者字符串开头的隐式 `B`）时的最小长度。
- `dp[i][j][0]`：表示生成子串 $S[i...j]$，**该区间内部可以包含新的 `B`** 时的最小长度（即全局的最优子结构）。

### 2. 状态转移方程

区间 DP 的常规做法是**先枚举区间长度 `len`，再枚举左端点 `i`**。

#### A. 计算 `dp[i][j][1]` (内部不新加 B)
既然不加 `B`，那么要么直接在末尾追加一个字符，要么利用 `R` 将前半段翻倍（如果前后两半相同）。
1. **直接追加字符**：
   $$dp[i][j][1] = dp[i][j-1][1] + 1$$
2. **使用 R 折叠（翻倍）**：
   如果当前区间长度为偶数，且前半段完全等于后半段（即 `S[i...mid] == S[mid+1...j]`）：
   $$dp[i][j][1] = \min(dp[i][j][1], dp[i][mid][1] + 1)$$
   *注：加 1 是因为我们需要放一个 'R' 字符。*

#### B. 计算 `dp[i][j][0]` (内部可以加 B)
此时我们可以枚举区间内**最后一个 `B`** 的出现位置。假设最后一个 `B` 放置在 $k+1$ 这个位置，那么整个区间会被分为前后两部分：
- 前半部分 $S[i...k]$：它的内部依然可以包含其他的 `B`，所以长度为 `dp[i][k][0]`。
- 后半部分 $S[k+1...j]$：因为 $k+1$ 是最后一个 `B`，所以这段子区间的内部不能再有其他的 `B` 了，因此长度为 `dp[k+1][j][1]`。
- 在 $k+1$ 处放置一个 `B` 的代价是 1。

1. **基础情况（内部实际上没加 B）**：
   $$dp[i][j][0] = dp[i][j][1]$$
2. **枚举断点 $k$ ($i \le k < j$)**：
   $$dp[i][j][0] = \min_{k} \left( dp[i][k][0] + dp[k+1][j][1] + 1 \right)$$

### 3. 初始化与答案
- 初始化：将 `dp` 数组所有元素设为无穷大（如 `0x3f3f3f3f`）。
- 边界：当 `len == 1` 时，生成单一字符无论是 `dp[i][j][0]` 还是 `dp[i][j][1]`，代价都为 1。
- 最终答案：整体字符串生成的最快方式，即 `dp[0][n-1][0]`。（因为如果第一个字符也是 B 可以省略，这恰好等效于全局的最优解结构中无需额外为开头加 B 代价）。

## 复杂度
- 时间复杂度：$O(N^3)$。枚举长度 $O(N)$，枚举左端点 $O(N)$，枚举断点 $k$ 以及使用 `substr` 比较均需要 $O(N)$ 时间。因为 $N \le 105$，总运算量大约 $10^6$ 级别，完全可以通过。
- 空间复杂度：$O(N^2)$。需要一个 $N \times N \times 2$ 的 DP 表，空间非常充裕。
