# Lab5B 最小代价经理问题说明

## 题意

- 有 n 个人，m 条有向边 s->t，费用为 C。
- 选择一位作为管理者，使得从他能到达所有人（形成以该点为根的有向生成树），总代价最小。
- 输出最小费用和管理者编号；对每个人输出其消息来源（管理者只输出自己）。

## 算法思路

- 本质是求有向最小生成树（最小入弧树，Arborescence）。
- 使用 Edmonds/Chu-Liu 算法：
  1) 每个点选择最小入边；若有点（除根）无入边则不可能；
  2) 根据入边找环，若无环则得到答案；
  3) 若有环则缩点，对跨环边减去对应入边权，递归求解，再展开父指针。
- 为避免枚举根，只跑一次：
  - 增加超级根 super，对每个节点连 super->i，权值 penalty = (max_w + 1) * (n + 1)；
  - 算法结束后，恰有一条 super 出边被选；若为 0 或 >1 条则 impossible；
  - 答案费用 = 总费用 - penalty。

## 复杂度

- 时间 O(VE)，空间 O(V+E)。这里 V = n + 1（含超级根），E = m + n。

## 实现要点

- 全部费用与累加使用 long long；penalty 截断到 INF/4 防止溢出。
- 递归保存 pre_edge 以恢复父节点；parent[i] 即消息来源。
- super 出边指向的节点即最终管理者，其父输出省略。

## 本次修复记录

- 修复了朱刘算法在“缩点后还原父指针”时的一个关键问题：
  - 原实现对**单点分量**默认使用 `pre_edge[node]` 还原父边；
  - 但当该分量在上层递归中被选中了一条来自其他分量的入边时，应优先使用该入边进行还原；
  - 否则会把可行解还原坏，可能错误输出 `impossible`。
- 统一了单点分量与环分量的“入边选择”逻辑：按上层 `sub_par[cid]` 对应父分量在当前图中选择最优进入边。

## 回归验证

- 样例通过。
- 小规模随机暴力对拍通过（逐例比对最优 `(cost, root)`）。
- 极端数据（同权完全图、零权环、大权值、稀疏/稠密随机）对拍通过。

## 自测建议

- 小图手算比对总代价与父节点。
- 不连通图（某点无入边）应输出 impossible。
- 多个可能根但费用不同，确认选择费用最小且唯一管理者。
- 含正环/负环/双向边，确认缩点后父指针恢复正确。
