               Selecting Breakpoints: Greedy Algorithm
   • Truck driver’s algorithm. Go as far as you can before refueling.
                  Sort breakpoints so that: 0 = b0 < b1 < b2 < ... < bn = L
                  S ← {0}       selected breakpoints
                  x ← 0         current location
                  while (x ≠ bn)
                     let p be largest integer such that bp ≤ x + c
                     if (bp = x)
                        return "no solution"
                     x ← bp
                     S ← S ∪ {p}
                  return S


   • Time complexity. O(n log n)
             Ø sorting breakpoints: O(n log n)
             Ø selecting all breakpoints: O(n)

                     Sort coin denominations: c1 < c2 < … < cn.

               Greedy Algorithms
   • Build up a solution in small steps.

   • Choose a decision at each step myopically to optimize some underlying
     criterion.

   • May not produce an optimal solution.

   • But can yield locally optimal solutions that approximate a globally optimal
     solution in a reasonable amount of time.

               Interval Par;;oning: Proof of Correctness
   • Observation. Greedy algorithm never schedules two incompatible
     lectures in the same classroom.
   • Theorem. Greedy algorithm is optimal.
   • Pf. Let d = number of classrooms that the greedy algorithm allocates.
             Ø Classroom d is opened because the greedy algorithm needed to schedule
               a lecture, say j, that is incompatible with all d – 1 other classrooms.
             Ø The d – 1 last lectures in those d – 1 classrooms each finish after sj.
             Ø Since the greedy algorithm sorted lectures by starting time, all those d – 1
               incompatible lectures start no later than sj.
             Ø Thus, we have d lectures overlapping at time sj + ε (i.e., right after sj).
             Ø Key observation: all schedules must use ≥ d classrooms. ▪

   • Theorem. Greedy schedule S is optimal.
   • Pf. Define S* to be an optimal schedule, and let’s see what happens.
             Ø Can assume S* has no idle time.
             Ø If S* has no inversions, then S = S*.

               Optimal Prefix Codes: Huffman Codes
   • Claim. There is an optimal prefix code with binary tree T* where the two
     lowest-frequency letters are assigned to leaves that are siblings in T*.
             Ø It suffices to find such an optimal tree.
   • Greedy approach: [Huffman 1952] Create tree bottom-up.
             Ø Make two sibling leaves for two lowest-frequency letters y and z.
             Ø View their common parent as a “meta-letter” ω whose frequency
               is the sum of frequencies of y and z.
             Ø Recursively build tree by replacing y and z with ω.
             Ø Finally, “open up” the meta-letter ω back to y and z.
   • Q. Why do we assign the sum of lowest frequencies to the meta-letter?
   • A. To make sure the reduced problem is well-shaped (sum of freqs = 1).

               Huffman’s Algorithm for Huffman Codes
   • Huffman’s algorithm: (produces a Huffman code)
              Huffman(S) {
                 if (|S| == 2)
                    return tree with root and 2 leaves
                 else
                    let y and z be lowest-frequency letters in S
                    let S’ be S with y and z removed
                    insert new letter ω in S’ with fω = fy + fz
                    T’ ← Huffman(S’)
                    T ← add two children y and z to leaf ω of T’
                    return T
              }
