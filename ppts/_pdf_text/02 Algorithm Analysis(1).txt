               Brute Force vs Polynomial Running Time
   • Brute force. For many non-trivial problems, there is a natural brute force
     search algorithm that checks every possible solution.
             Ø Typically takes 2n steps (or worse) for inputs of size n.
             Ø Unacceptable in practice.

   • Desirable scaling property. If the input size is increased by a constant
     factor C1, the algorithm should also slow down by a constant factor C2.
   • Def. An algorithm is polynomial-time if there exist constants a > 0, b > 0
     such that the running time of the algorithm is bounded by anb.
             Ø Note that the above scaling property holds for polynomials, e.g., C2 = C1b.


                                                                                             10
Slides by Kevin Wayne. Copyright © 2005 Pearson-Addison Wesley. All rights reserved.
               Polynomial Running Time
   • We say that an algorithm is efficient if has a polynomial running time.
   • Theory. Definition is (relatively) insensitive to models of computation.
   • Practice. It really works!
             Ø The polynomial-time (e.g., anb) algorithms that people develop almost always
               have both small constant a and small exponent b.
             Ø Breaking through the exponential barrier (e.g., 2n) of brute force typically
               exposes some crucial structure of the problem.
   • Exceptions:
             Ø Some polynomial-time algorithms in the wild have galactic constants and/or
               huge exponents, then useless in practice, e.g., 6.02 x 1023 x n20.
             Ø Some exponential-time (or worse) algorithms are widely used because the
               worst-case instances (i.e., inputs) seem to be rare, e.g., the simplex method.

               Worst-Case/Average-Case Analysis
   • Worst-case analysis. Worst possible running time on any input of size n.
             Ø Generally captures efficiency in practice.
             Ø Draconian view, but hard to find effective alternative.

   • Average-case analysis. Average running time on random input of size n.
             Ø Hard (or impossible) to accurately model real-world instances with random
               distributions, e.g., the real-world instances may not follow the distribution.
             Ø Algorithms tuned for a certain distribution of instances may perform poorly on
               other distributions.

2. Asymptotic Order of Growth


                                14
               Notation
   • Let function T(n) denote the worse-case running time of an algorithm on
     any input of size n.
   • Upper bounds. T(n) is O(f(n)) if there exist constants c > 0 and n0 ≥ 0 such
     that for all n ≥ n0 we have 0 ≤ T(n) ≤ c · f(n).
   • Lower bounds. T(n) is Ω(f(n)) if there exist constants c > 0 and n0 ≥ 0 such
     that for all n ≥ n0 we have T(n) ≥ c · f(n) ≥ 0.
   • Tight bounds. T(n) is Θ(f(n)) if T(n) is both O(f(n)) and Ω(f(n)).
   • Example: T(n) = 32n2 + 17n + 32
             Ø T(n) is O(n2), O(n3), Ω(n2), Ω(n), and Θ(n2) .
             Ø T(n) is not O(n), Ω(n3), Θ(n), or Θ(n3).

Slides by Kevin Wayne. Copyright © 2005 Pearson-Addison Wesley. All rights reserved.
               Remark on the Use of Notation
    • Slight abuse of notation. T(n) = O(f(n))
             Ø = is not an equivalence relation:
               üf(n) = 5n3; g(n) = 3n2
               üf(n) = O(n3) = g(n)
               üf(n) ≠ g(n).
             Ø Better notation: T(n) ∈ O(f(n))

    • Vacuous statement example. Any compare-based sorting
      algorithm requires at least O(n log n) compares in the worst case.
             Ø The statement does not "type-check".
             Ø Should use Ω for lower bounds.

Slides by Kevin Wayne. Copyright © 2005 Pearson-Addison Wesley. All rights reserved.
               Properties
   • Reflexivity and constants:
             Ø f = O(f), f = Ω(f), f = Θ(f)
             Ø If c > 0 is a constant, then cf = O(f), cf = Ω(f), cf = Θ(f).
   • Transitivity:
             Ø If f = O(g) and g = O(h) then f = O(h).
             Ø If f = Ω(g) and g = Ω(h) then f = Ω(h).
             Ø If f = Θ(g) and g = Θ(h) then f = Θ(h).
   • Sums and Products:
             Ø If f = O(h) and g = O(h), then f + g = O(h) and f · g = O(h2).
             Ø If f = Ω(h) and g = Ω(h), then f + g = Ω(h) and f · g = Ω(h2).
             Ø If f = Θ(h) and g = Θ(h), then f + g = Θ(h) and f · g = Θ(h2).

               Asymptotic Bounds for Some Common Functions
   • Polynomials. If ad > 0, then a0 + a1n + … + adnd = Θ(nd).
   • Polynomial time. The running time is O(nc) for some constant c > 0
     independent of the input size n.
   • Logarithms. O(loga n) = O(logb n) = O(log n) for any constants a, b > 1.
                                                                                       no need to specify log base

   • Logarithms < polynomials. For every a > 1, c > 0, loga n = O(nc).
   • Exponentials > polynomials. For every r > 1, c > 0, nc = O(rn).
   • Factorials. n! = 2Θ(n log n), i.e., log2 n! = Θ(n log n).

   • Merge. Combine two sorted lists A = a1, a2, …, an and B = b1, b2,… ,bn into

                               i = 1, j = 1

               Cubic Time: O(n3)
   • Cubic time. Enumerate all triples of elements.
   • Set disjointness. Given n sets S1, …, Sn, each of which is a subset of
     1, 2, …, n, is there some disjoint pair of these sets?
             Ø O(n3) solution: for each pairs of sets, check if they are disjoint.
                               foreach set Si {
                                  foreach other set Sj {
                                      foreach element p of Si {
                                        determine whether p also belongs to Sj
                                      }
                                      if (no element of Si belongs to Sj)
                                        report that Si and Sj are disjoint
                                  }
                               }

               Polynomial Time: O(nk)
   • Independent set of size k. Given a graph and a constant k, are there k
     nodes such that no two nodes are joined by an edge?
             Ø O(nk) solution: enumerate all subsets S of k nodes.
                           foreach subset S of k nodes {
                              check whether S in an independent set
                              if (S is an independent set)
                                 report S is an independent set
                           }

             •        Check whether S is an independent set is O(k2)
             •        Number of k element subsets is O(nk / k!)
             •        Overall: O(k2 nk / k!) = O(nk)
                                                                                       still polynomial time, e.g., for k = 17, but not practical

   • Independent set of maximum size. Given a graph, what is maximum size
     of an independent set?
             Ø O(n2 2n) solution: Enumerate all subsets.

                            check whether S in an independent set

                               update S* = S
                         }

   Factorial Time: O(n!)
• Factorial time. Enumerate all permutations of length n.

• Traveling salesman problem (TSP). Given a set of n cities, with distances
  between all pairs, what is the shortest tour that visits all cities?
   Ø O(n!) solution: search all (n – 1)! tours (starting and ending at the same city)

   Five Representative Problems: Summary
• Interval scheduling: O(n log n) greedy algorithm.
• Weighted interval scheduling: O(n log n) dynamic programming
  algorithm.
• Bipartite matching: O(nk) network flow algorithm.
• Independent set: NP-complete. we will learn this in later sections
• Competitive facility location: PSPACE-complete.
