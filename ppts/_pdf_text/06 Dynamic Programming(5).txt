               Algorithmic Paradigms
   • Greedy. Process the input in some order, and myopically making
     irrevocable decisions to optimize some underlying criterion.

   • Divide-and-Conquer. Break up a problem into independent subproblems,
     solve each subproblem, and combine solution to subproblems to form
     solution to original problem.
                                                                                       fancy name for caching intermediate results for later use

   • Dynamic Programming. Break up a problem into a series of overlapping
     subproblems, combine solutions to smaller subproblems to form solution
     to large subproblem.

               Dynamic Programming: Binary Choice
   • Def. OPT(j) = max weight of any subset of mutually compatible jobs for
     subproblem consisting only of jobs 1, 2, ..., j.
   • Goal. OPT(n) = max weight of any subset of mutually compatible jobs.

   • Bellman equation:
   • Pf. (by cases)
             Ø Case 1: OPT(j) does not select job j.
               üMust be optimal solution to problem consisting of remaining jobs 1, 2, ..., j – 1.
             Ø Case 2: OPT(j) selects job j.
               üCollect weight wj. (Then can’t use incompatible jobs p(j) + 1, p(j) + 2, ..., j – 1.)
               üMust include optimal solution to problem consisting of remaining compatible
                jobs 1, 2, ..., p(j).

               Weighted Interval Scheduling: Algorithm
   • Top-down DP algorithm (memorization). Cache results of subproblem j
     in M[j] to avoid solving the same subproblem more than once.
                Input: n, s1,…,sn , f1,…,fn , w1,…,wn

                Sort jobs by finish times and renumber so that f1 £ f2 £ ... £ fn.
                Compute p(1), p(2), …, p(n) via binary search

                M[0] = 0                           M[] is global array

                return M-Compute-Opt(n)

                M-Compute-Opt(j) {              recur only when M[j] is unknown
                   if (M[j] is uninitialized)
                      M[j] = max{ M-Compute-Opt(j - 1), wj + M-Compute-Opt(p(j)) }
                   return M[j]
                }

               Weighted Interval Scheduling: Running Time
   • Theorem. Top-down DP algorithm takes O(n log n) time.
   • Pf. (direct proof)
             Ø Sort by finish time: O(n log n).
             Ø Compute p(×) for each j: O(n log n) via binary search.
             Ø M-Compute-Opt(j): each invocation takes O(1) time and either
               1) returns an existing value M[j]; or
               2) initializes M[j] and makes two recursive calls
             Ø At most n uninitialized M[] entries ⇒ at most 2n recursive calls.
             Ø Overall running time of M-Compute-Opt(n) is O(n). ▪

               Weighted Interval Scheduling: Finding a Solution
   • Q. DP computes optimal value M[n]. How to find an optimal solution?
   • A. Do backtrack post-processing by calling Find-Solution(n).
                     Find-Solution(j) {
                        if (j = 0)
                           return empty set
                        else if (M[j] > M[j - 1])
                           return {j} ∪ Find-Solution(p(j))                            M[j] = max{ M[j - 1], wj + M[p(j)] }
                        else
                           return Find-Solution(j - 1)
                     }

   • Running time. Number of recursive calls ≤ n ⇒ O(n)
   • Note. Such a trace-back technique also applies to other DP algorithms to
     find an optimal solution to other problems.

               Weighted Interval Scheduling: Bottom-Up
   • Bottom-up DP algorithm. Unwind recursion.
             Ø Usually faster than top-down due to no overhead from recursive function calls.
               Input: n, s1,…,sn , f1,…,fn , v1,…,vn

               Sort jobs by finish times and renumber so that f1 £ f2 £ ... £ fn.
               Compute p(1), p(2), …, p(n) via binary search

               M[0] = 0                 previously computed M[] values
               for j = 1 to n
                   M[j] = max{ M[j - 1], vj + M[p(j)] }

               return M[n]

   • Running time. Bottom-up DP takes O(n log n) time.

               Knapsack Problem: Algorithm
   • Bottom-up DP algorithm:
                Input: n, W, w1,…,wn, v1,…,vn

                for w = 0 to W
                   M[0, w] = 0

                for i = 1 to n
                   for w = 1 to W
                      if (wi > w)
                         M[i, w] = M[i - 1, w]
                      else
                         M[i, w] = max { M[i - 1, w], vi + M[i - 1, w - wi ] }

                return M[n, W]

   • Running time. O(nW)

   Knapsack DP Algorithm: Space Complexity
• Bottom-up DP algorithm:
   Input: n, W, w1,…,wn, v1,…,vn

   for w = 0 to W
      M[0, w] = 0

   for i = 1 to n
      for w = 1 to W
         if (wi > w)                      only M[i – 1, .] is used!
            M[i, w] = M[i - 1, w]
         else
            M[i, w] = max { M[i - 1, w], vi + M[i - 1, w - wi ] }

   return M[n, W]

• Space. O(nW)       can we reduce space? 1010 operations (~100s) vs 1010 bytes (~10G)

   Knapsack DP Algorithm: Space Complexity
• Bottom-up DP algorithm:
   Input: n, W, w1,…,wn, v1,…,vn

   for w = 0 to W
      M[w] = 0         1-D scrolling array

   for i = 1 to n
      for w = W to 1      updating in reverse order
         if (wi > w)
            M[w] = M[w]         these M[.] values are from (i – 1)-th iteration!
         else
            M[w] = max { M[w], vi + M[w - wi ] }

   return M[W]

• Space. O(W)

   • Theorem. Hirschberg’s algorithm uses Θ(m + n) space.
   • Pf. (direct proof)
             Ø        Each recursive call uses Θ(m) space to compute f(·, n/2) and g(·, n/2).

             Ø        Θ(1) space per recursive call to save the dividing node.
             Ø        Number of recursive calls ≤ n since there are ≤ n dividing nodes.
             Ø        Together, the algorithm takes up Θ(m + n) space. ▪

   • Theorem. Let T(m, n) = max running time of Hirschberg’s algorithm on

             Ø successor[v] = next node on a v-t path (used to find shortest paths)
   • Remark. This space-efficient algorithm is called Bellman-Ford-Moore.

   • Performance optimization. If d[w] was not updated in iteration i – 1,
     then no need to consider edges entering w in iteration i.
   • Remark. This time-efficient implementation of Bellman-Ford-Moore,
     together with a (first-in-first-out) queue maintaining only updated nodes,
     is also known as the shortest path faster algorithm (SPFA).
