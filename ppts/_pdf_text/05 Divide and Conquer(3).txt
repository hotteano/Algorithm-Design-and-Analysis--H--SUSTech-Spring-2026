               Divide-and-Conquer Recurrences
   • Divide-and-Conquer recurrences. T(n) = aT(n/b) + f(n) with T(1) = Θ(1)

   • Master Theorem. Let a ≥ 1, b ≥ 2, c ≥ 0 and suppose T(n) is a function on
     the non-negative integers that satisfies the recurrence
                             T(n) = aT(n/b) + Θ(nc)
      with T(1) = Θ(1), where n/b means either ⌊n/b⌋ or ⌈n/b⌉. Then,
       Ø Case 1: If c > logb a, then T(n) = Θ(nc).
       Ø Case 2: If c = logb a, then T(n) = Θ(nc log n).
       Ø Case 3: If c < logb a, then T(n) = Θ(nlogb a).

   • Sort-and-Count algorithm:
                Sort-and-Count(L) {
                   if (list L has one element) return (0, L)

                         Divide the list into two halves A and B
                            (rA, A) ← Sort-and-Count(A)
                            (rB, B) ← Sort-and-Count(B)
                            (rAB, L) ← Merge-and-Count(A, B)

                         return (rA + rB + rAB, L)
                }

   • Merge-and-Count sub-algorithm:
             Ø Input: A and B are sorted (by Sort-and-Count).
             Ø Output: L is sorted.

               Closest Pair of Points: O(n log n) Algorithm
   • Divide-and-Conquer algorithm: (sort by x-axis and y-axis beforehand)
             Closest-Pair(p1, ..., pn) {
                Compute vertical line L such that half the points
                are on each side of the line and Partition the points.                 O(n)     use x-sorted list

                     δ1 ← Closest-Pair(left half)
                     δ2 ← Closest-Pair(right half)                                     2T(n / 2)
                     δ ← min(δ1, δ2)

                     Delete all points further than δ from line L.                     O(n)

                     Sort remaining points by y-coordinate.                            O(n log n) use y-sorted list
                     Scan points in y-order and compare distance between
                     each point and next 7 neighbors. If any of these                  O(n)
                     distances is less than δ, update δ.

                     return δ
             }

               Karatsuba Algorithm
   • Karatsuba algorithm: (for integers longer than one 32/64-bit word)
             Karatsuba-Multiply(x, y, n) {
                if (n = 1) return x * y
                else
                   m ←⌈ n / 2 ⌉
                   a ← ⌊ x / 2m ⌋; b = x mod 2m                                        O(n)
                   c ← ⌊ y / 2m ⌋; d = y mod 2m
                   e ← Karatsuba-Multiply(a, c, m)
                   f ← Karatsuba-Multiply(b, d, m)                                     3T(n / 2)
                   g ← Karatsuba-Multiply(|a – b|, |c – d|, m)
                   Flip sign of g if needed

                            return 22m e + 2m(e + f – g) + f                           O(n)
             }


   • Practice. Faster than grade-school for integers longer than ≈320 bits.

               Fast Fourier Transform (FFT)
   • Goal. Evaluate a degree n – 1 polynomial A(x) = a0 + ... + an – 1 xn – 1 at its
     all nth roots of unity: ω0, ω1, …, ωn – 1.
   • Divide. Break up polynomial into even and odd powers.
             Ø        Aeven(x) = a0 + a2 x + a4 x2 + … + an – 2 x ½n – 1
             Ø        Aodd (x) = a1 + a3 x + a5 x2 + … + an – 1 x ½n – 1
             Ø        A(x) = Aeven(x2) + x Aodd(x2)
             Ø        A(–x) = Aeven(x2) – x Aodd(x2)
   • Conquer. Evaluate Aeven(x), Aodd(x) at ½nth roots of unity: ν0, ν1, …, ν ½n – 1
   • Combine. (Note that ν = ω2, ν½ n = 1, ω½n = –1 and –ωk = ωk + ½n.)
             Ø A(ωk) = Aeven(νk) + ωk Aodd(νk), 0 ≤ k < n/2
             Ø A(–ωk) = Aeven(νk) – ωk Aodd(νk), 0 ≤ k < n/2

               Integer Multiplication Revisited
   • Integer Multiplication. Given two n bit integers a = an – 1 … a1a0 and
     b = bn – 1 … b1b0, compute their product c = ab.
   • Convolution algorithm:
             Ø        Form two polynomials A(x), B(x).
             Ø        Note that a = A(2), b = B(2).
             Ø        Compute C(x) = A(x) B(x) and evaluate C(2) = ab.
             Ø        Running time: O(n log n) floating-point operations (O(n log3 n) bit operations).
   • Optimization: [Schönhage-Strassen 1971]
             Ø O(n log2 n) bit operations over complex numbers (with O(log n) bit precision)
             Ø O(n log n log log n) bit operations over ring of integers (mod a Fermat number)
   • Practice. GNU Multiple Precision Arithmetic Library (GMP) switches to
     FFT-based algorithms when n is large (≥ ≈5000).

               FFT in Practice
   • FFT in the West (FFTW): [Frigo-Johnson]
             Ø        Optimized C library.
             Ø        Features: DFT, DCT, real, complex, any size, any dimension.
             Ø        Won 1999 Wilkinson Prize for Numerical Software.
             Ø        Portable, competitive with vendor-tuned code.
   • Implementation details:
             Ø Core algorithm is an in-place, non-recursive version of Cooley-Tukey.
             Ø Instead of executing a fixed algorithm, it evaluates the hardware and uses a
               special-purpose compiler to generate an optimized algorithm catered to the
               “shape” of the problem.
             Ø Runs in O(n log n) time, even when n is prime.
             Ø Multidimensional FFTs.
